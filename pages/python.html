Python & Flask

Sanal ortam kurulumu, paket yönetimi ve Gunicorn ile production deploy

Sanal Ortam (venv)
venv oluştur (Linux)
python3 -m venv env
Proje klasöründe sanal ortam oluştur. Bir kez yapılır.
venv aktif et (Linux)
source env/bin/activate
Sanal ortamı aktif et. Terminal başında (env) görünür.
venv aktif et (Windows)
.\env\bin\activate
Windows PowerShell'de sanal ortamı aktif et.
venv deaktif et
deactivate
Sanal ortamdan çık.
Paket Yönetimi
Flask projesi için paketler kur
pip install flask authlib python-dotenv gunicorn requests
Dokushoka projesinde kullanılan tüm paketleri kur.
requirements.txt oluştur
pip freeze > requirements.txt
Kurulu paketleri dosyaya kaydet. GitHub'a push etmeden önce güncelle.
requirements.txt'ten kur
pip install -r requirements.txt
Yeni ortamda tüm paketleri tek komutla kur.
Gunicorn
uygulamayı başlat (test)
gunicorn --bind 0.0.0.0:5000 app:app
Uygulamayı Gunicorn ile çalıştır. Erişim için port 5000 UFW'de açık olmalı.
timeout ile başlat
gunicorn --bind 0.0.0.0:5000 --timeout 30 app:app
Worker timeout'unu 30 saniyeye ayarla. Donma sorunlarında kullanılır.
.env Kullanımı
app.py başına ekle
from dotenv import load_dotenv import os load_dotenv() app.secret_key = os.getenv("SECRET_KEY")
.env dosyasındaki değerleri okuma. API key'leri kod içine yazma.
