Git & GitHub

Repo başlatmadan push'a, güncellemeye kadar tüm Git workflow'u

İlk Kurulum
kimlik tanımla
git config --global user.email "burakhankirlak@outlook.com" git config --global user.name "HanMiyazaki"
Git commitleri için kimlik bilgisi tanımla. Bir kez yapılır.
execution policy (Windows)
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
PowerShell'de script çalıştırmayı etkinleştir. Windows'ta bir kez yapılır.
Yeni Repo
repo başlat
git init
Mevcut klasörü Git reposuna dönüştürür.
remote ekle & ilk push
git remote add origin https://github.com/HanMiyazaki/REPO.git git branch -M main git add . git commit -m "initial commit" git push -u origin main
Yeni oluşturulan GitHub reposuna bağlan ve ilk kodu gönder.
Günlük Kullanım
değişiklikleri hazırla & gönder
git add . git commit -m "mesaj buraya" git push
Tüm değişiklikleri GitHub'a gönder. Her güncelleme sonrası kullanılır.
VPS'e çek
git pull
GitHub'daki son değişiklikleri VPS'e indir.
durum kontrol
git status
Hangi dosyaların değiştiğini, hangilerinin staged olduğunu göster.
commit geçmişi
git log --oneline
Önceki tüm commitleri kısa formatta listele.
zorla push (dikkatli kullan)
git push -u origin main --force
dikkat
Remote'u lokal ile ezer. Conflict durumlarında kullan, ekip projesinde tehlikeli.
.gitignore — Dışarıda tutulacaklar
standart Python .gitignore
# .gitignore içeriği .env venv/ env/ __pycache__/ *.pyc database.db
Bu dosyalar GitHub'a gönderilmez. API key'ler ve veritabanı güvende kalır.
