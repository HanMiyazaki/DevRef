Nginx

Config yönetimi, test, reload ve reverse proxy kurulumu

Temel Komutlar
config test
nginx -t
Config dosyasında syntax hatası olup olmadığını kontrol et. Her değişiklikten önce çalıştır.
test & reload (güvenli)
nginx -t && systemctl reload nginx
önerilen
Önce test eder, başarılıysa yeniden yükler. Her config değişikliğinde bunu kullan.
yeniden başlat
systemctl restart nginx
Nginx'i tamamen durdurur ve yeniden başlatır. Kısa süre kesinti olabilir.
durum kontrol
systemctl status nginx
Nginx'in çalışıp çalışmadığını ve son logları göster.
Site Yönetimi
site aktif et
ln -s /etc/nginx/sites-available/site /etc/nginx/sites-enabled/
sites-available'daki config'i sites-enabled'a bağla. Sembolik link oluşturur.
site deaktif et
rm /etc/nginx/sites-enabled/site
Siteyi devre dışı bırak. sites-available'daki dosyaya dokunmaz.
mevcut config'i düzenle
nano /etc/nginx/sites-available/site
Config dosyasını nano editörde aç. Kaydetmek için CTRL+X → Y → Enter.
Config Şablonları
statik site config
server { listen 80; server_name domain.com; root /var/www/klasor; index index.html; location / { try_files $uri $uri/ =404; } }
HTML/CSS/JS statik site için temel config. Certbot sonrası HTTPS otomatik eklenir.
Flask reverse proxy config
server { listen 80; server_name domain.com; location / { proxy_pass http://127.0.0.1:5000; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; } }
Flask/Gunicorn uygulaması için reverse proxy config. Port numarasını uygulamana göre değiştir.
