Dokushoka Deploy

Değişiklik yaptıktan sonra canlı ortama alma süreci

ℹ
Canlı URL: https://dokushoka.burakkirlak.info
Kod dizini: /var/www/dokushoka
Servis adı: dokushoka
Güncelleme — Her değişiklik sonrası
1. Lokal değişiklikleri GitHub'a gönder
git add . git commit -m "değişiklik açıklaması" git push
Lokalde yapılan değişiklikleri önce GitHub'a gönder.
2. VPS'e çek ve servisi yeniden başlat
cd /var/www/dokushoka && git pull && systemctl restart dokushoka
ana komut
SSH'ta çalıştır. GitHub'daki son kodu çeker, servisi yeniden başlatır. Tek satır, hızlı.
Servis Yönetimi
servis durumu
systemctl status dokushoka
Servisin çalışıp çalışmadığını ve son logları göster.
hata logları
journalctl -u dokushoka -n 50
Son 50 log satırını göster. Hata durumunda buraya bak.
manuel başlat (test için)
systemctl stop dokushoka cd /var/www/dokushoka && source env/bin/activate gunicorn --bind 0.0.0.0:5000 app:app
Servisi durdurup terminalden başlat. Hataları canlı izlemek için kullanılır.
Admin Yönetimi
kullanıcıya admin yetkisi ver
cd /var/www/dokushoka sqlite3 database.db # SQLite açıldıktan sonra: UPDATE users SET is_admin = 1 WHERE email = 'email@domain.com'; # Çıkmak için CTRL+D
Belirtilen e-posta adresine admin yetkisi ver. Kitap ekleme/düzenleme yetkisi kazanır.
tüm kullanıcıları listele
sqlite3 database.db SELECT id, email, is_admin FROM users;
Kayıtlı tüm kullanıcıları ve admin durumlarını göster.
